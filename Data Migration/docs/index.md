# PostgreSQL 手动迁移助手开发文档

本文档用于设计一个面向 PostgreSQL 到 PostgreSQL 的手动迁移辅助工具。当前阶段不负责自动迁移数据，也不自动创建 publication / subscription；重点是辅助已经手动完成结构迁移、并准备使用原生逻辑复制同步数据的场景。

当前优先级聚焦在两个可快速落地、又能显著降低切换风险的能力：

- 一键设置目标库序列值
- 数据检查与迁移前后校验

## 产品目标

构建一个 PostgreSQL 手动迁移助手，帮助用户在切换前完成风险检查、序列校准和数据一致性检查。

第一阶段不追求完整替代专业迁移平台，而是先解决迁移中最容易遗漏、但影响切换成功率的关键问题。自动创建发布、订阅和自动迁移会放到后续版本。

## 第一阶段范围

第一阶段建议围绕以下能力实现：

1. PostgreSQL 实例连接与保存
2. 源库和目标库基础信息扫描
3. `REPLICA IDENTITY` 模式展示
4. 源库序列值采集
5. 目标库序列一键设置
6. 表级数据检查
7. 迁移准备任务状态与操作日志

## 暂不纳入第一阶段

以下能力可以规划，但不建议放进 MVP：

- 自动创建 publication / subscription
- 自动迁移数据
- 自研逻辑复制消费与 apply
- 复杂数据转换
- 跨数据库类型迁移
- 自动 DDL 同步
- 双向复制
- 自动回滚
- 完整权限迁移

## 推荐迁移路线

第一阶段优先使用 PostgreSQL 原生能力：

- 使用 `pg_dump --schema-only` 迁移结构
- 手动使用 `CREATE PUBLICATION` 创建源库发布
- 手动使用 `CREATE SUBSCRIPTION` 创建目标库订阅
- 使用系统 catalog 扫描表、索引、序列和复制配置
- 使用批量 SQL 完成 sequence 校准
- 使用行数、主键范围和抽样校验做数据检查

## 核心流程

```mermaid
flowchart LR
  A["手动完成结构迁移"] --> C["创建迁移准备任务"]
  C --> D["表与 Replica Identity 检查"]
  D --> B["手动配置逻辑复制"]
  B --> E["等待数据同步追平"]
  E --> F["切换前数据检查"]
  F --> G["预览并设置目标库序列"]
  G --> H["切换前复核"]
```
