# 前端控制台

前端位于 `frontend/` 目录，使用 Vite + React + TypeScript + Ant Design。

第一版控制台包含：

- 实例管理
- 代理连接配置
- 连接测试结果展示
- 迁移准备任务列表
- 准备详情
- 独立切换前检查页
- 切换准备度摘要
- 源库表扫描与任务表选择
- 序列预览与一键设置
- 数据检查
- 数据检查临时表选择
- 表元数据与 Replica Identity 风险查看
- 操作日志查看

## 启动方式

第一次安装依赖：

```bash
cd frontend
HTTP_PROXY=http://127.0.0.1:8001 HTTPS_PROXY=http://127.0.0.1:8001 npm install
```

启动前端：

```bash
cd frontend
scripts/start-frontend.sh
```

默认地址：

```text
http://127.0.0.1:5174
```

## 后端依赖

前端会通过 Vite dev server 把 `/api` 和 `/health` 代理到后端：

```text
http://127.0.0.1:8000
```

后端需要先启动：

```bash
cd backend
scripts/start-api.sh
```

## 页面说明

### 实例管理

用于新增、删除和测试 PostgreSQL 实例连接。新增实例时可以配置：

- 基础连接信息
- SSL mode
- HTTP CONNECT / SOCKS4 / SOCKS5 代理

### 迁移准备

用于创建迁移准备任务，并进入准备详情页执行逻辑复制前检查。当前页面不会自动迁移数据，也不会自动创建 publication / subscription。

创建准备任务时可以：

- 选择源库和目标库。
- 填写 schema 范围。
- 扫描源库表并多选数据检查表。
- 在无法扫描时手动填写 `schema.table` 列表。

准备详情包含：

- 准备任务基础信息
- 源库和目标库实例名称
- 逻辑复制准备度摘要，包括检查失败数和 Replica Identity 风险
- 数据检查
- 数据检查临时表选择，不需要重新创建准备任务
- 表元数据，包括 Replica Identity 风险列
- 表元数据分页大小可调整，最多每页 1000 行
- 可从合格唯一索引中选择并设置 `REPLICA IDENTITY USING INDEX`
- 操作日志
- 数据检查 JSON 详情展开查看

准备任务列表支持删除任务。删除任务会同时清理该任务的操作日志、序列结果和数据检查结果。

### 切换前检查

切换前检查是独立页面，用于聚合一个准备任务的切换准备情况。它不会替代准备详情，而是专门回答“现在能不能切换”。

推荐使用顺序：

1. 手动完成结构迁移。
2. 在迁移准备任务中执行表与 Replica Identity 检查，确保逻辑复制前置条件可用。
3. 手动配置并启动逻辑复制或其他数据同步方案。
4. 数据同步完成后预览并设置目标库序列。
5. 在切换前检查页复核阻断项、警告项和操作日志。

自动创建发布、订阅和自动迁移数据会放到后续版本。

页面包含：

- 任务选择器
- 源库和目标库摘要
- 切换建议：可切换、需确认、不可切换
- 阻断项和警告项统计
- 序列待设置和跳过数量
- 序列步进值设置
- 源端当前值、目标端当前值和预设目标值
- 数据检查失败数量
- Replica Identity 风险数量
- 必处理事项列表
- 表与 Replica Identity 明细
- 表与 Identity 明细分页大小可调整，最多每页 1000 行
- 表与 Identity 明细支持选择可用唯一索引并设置 `USING INDEX`
- 数据检查明细
- 序列预览、一键设置和明细
- 操作日志

当前切换前检查使用已有任务结果进行汇总。没有执行过序列预览、数据检查或表元数据扫描时，会显示为 warning，避免误判为可切换。序列预览和一键设置在这个页面执行，因为它们通常发生在数据同步追平之后、业务切换之前。
